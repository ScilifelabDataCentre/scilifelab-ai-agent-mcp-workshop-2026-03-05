# Session 2 – AI Agent Collaboration with the Model Context Protocol (MCP)

> **SciLifeLab AI Agents in Life Sciences Workshop** · Stockholm · 2026-03-05

In Session 1 you built a drug discovery AI agent with LangGraph, a brain that could reason, plan, and call tools like `resolve_smiles`, `get_properties`, and `lit_search`. But those tools were locked inside your Python file.

**In this session you break them free.** You will expose those same tools over the Model Context Protocol (MCP) so that any compatible AI application can discover and call them, without touching your source code.

---

## How to use this material

### Your main entry point

Open **`mcp_workshop.ipynb`** in Jupyter. This is the notebook you will work through during the session. It contains reading sections, code examples, and **11 fill-in-the-blank gaps** marked with `# TODO` and `# ... YOUR CODE HERE`.

### If you get stuck

**`mcp_workshop_answers.ipynb`** contains the completed implementations for every gap. Try the exercises yourself first, use the answers only as a reference after you have attempted each gap.

### How the notebooks work

The notebooks generate Python files (`basic_server.py`, `basic_client.py`, etc.) using `%%writefile` cells. You edit the code **inside the notebook cells**, then run the cell to write the file to disk. You then run those files in separate terminal windows.

## Getting started

### Docker (recommended)

The workshop runs inside a Docker container. See the **parent directory README** (`session-2-mcp/README.md`) for full Docker build and run instructions.

Once the container is running, open `http://localhost:7860` in your browser and navigate to `1-mcp-from-scratch/mcp_workshop.ipynb`.

### API key

Your `.env` file should contain:

```
OPENAI_API_KEY="paste-your-key-here"
```

If you started the Docker container with `-e OPENAI_API_KEY="sk-..."`, this is written automatically. Otherwise, edit the `.env` file in this directory.

## Running the servers and clients

You will need **two terminal windows** inside the container (or two Jupyter terminal tabs) — one for the server, one for the client.

### Basic server + client (Part 3)

```bash
# Terminal 1: start the server
cd 1-mcp-from-scratch/
python basic_server.py          # → http://localhost:8501

# Terminal 2: run the client
cd 1-mcp-from-scratch/
python basic_client.py
```

### Advanced server + client (Part 5)

```bash
# Terminal 1: start the server (stop the basic server first)
python advanced_server.py       # → http://localhost:8502

# Terminal 2: run the client
python advanced_client.py
```

### LangGraph + MCP integration (Part 6)

Keep `advanced_server.py` running on port 8502. Run the Part 6 cells directly in the Jupyter notebook.

### SDK bonus (Part 7, optional)

```bash
# Terminal 1
cd 2-bonus-mcp-sdk-implementation/
python sdk_basic_server.py      # → http://localhost:8501

# Terminal 2
python sdk_basic_client.py
```

> See `2-bonus-mcp-sdk-implementation/README.md` for the full SDK comparison.

### Serve app integration bonus (Part 8, optional)


## Troubleshooting

| Symptom | Fix |
|---|---|
| `Address already in use` on a port | `lsof -ti:8501 | xargs kill` (replace 8501 with the relevant port) |
| `ModuleNotFoundError` for any package | All dependencies are pre-installed in the Docker image. If running locally, see `requirements.txt` in the parent directory |
| OpenAI 401 / authentication error | Check `OPENAI_API_KEY` in your `.env` file |
| RDKit not found | No action needed, properties fall back to pre-computed values in `drug_db.json` |
| `drug_db.json` not found | Run notebook cell § 3.1 first |
| Client shows `ConnectionError` | Make sure the corresponding server is running in another terminal |
| `%%writefile` cell shows no output | The file was already written, re-run the cell to overwrite |
